# Clean-Room Boundary

This repository is a public-safe rebuild of a playlist sorting concept.

It does not include:

- school starter code
- assignment instructions or grading rubrics
- private tests
- private datasets
- instructor-provided scaffolding
- teammate code
- copied files from academic repositories

The goal is to demonstrate the engineering skills behind the idea without exposing protected academic material.

## Rebuild Rules

1. Start from a blank repository.
2. Use public documentation only.
3. Write new file names, structure, and implementation choices.
4. Use mock data for the public demo.
5. Keep credentials in local browser storage or environment variables only.
6. Create sorted playlists as new playlists instead of mutating user data by default.

## Recruiter Signal

This project is meant to show:

- OAuth PKCE architecture
- external API integration
- sorting and filtering logic
- frontend state management
- privacy-aware product decisions
- testable pure functions
- CI-backed project hygiene
